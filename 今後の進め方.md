# 株価シグナル通知アプリ 実装仕様書【追補版】

> 本書は `stock-signal-app-spec.md`（基本仕様書）の **追補** です。
> 基本仕様書を前提とし、それを上書き・拡張する差分のみを記載します。
> 基本仕様書に書かれた前提（ローカル完結 / 日足スイング / 自動売買しない / 2プロセス構成 / スコアリング方式）はすべて引き継ぎます。

---

## A. この追補の目的

基本仕様書は「日足の指標をスコアリングする最小構成」だった。
これを **同じOHLCVデータの範囲内で多面的に使い**、判断の質と実用性を一段引き上げる。
新しい外部データ源は増やさない（決算日程・ニュース・板情報は引き続き対象外）。

追加する強化は次の4点。

1. **出来高フィルター** … シグナルのダマシを減らす
2. **週足トレンド足切り** … 逆張りの事故を減らす（マルチタイムフレーム）
3. **ATRによる出口自動設計** … 損切り・利確を機械的に算出
4. **翌日の指値提案（作戦ボード）** … 「いくらで売買するか」まで提示し、夜に翌日の注文を準備できるようにする

### 運用イメージ（このアプリが目指す使い方）

```
平日の夜（終値確定後・20時以降）
  ① 再判定を実行
  ② 「明日の作戦ボード」を確認（判定・提案指値・利確/損切ライン）
  ③ 証券アプリで指値注文を置く（発注は人間）
日中
  ④ 指値に刺されば約定（画面に張り付かない）
  ⑤ 利確/損切も指値で置いておけば自動決済
```

監視は**数銘柄をじっくり**が前提。全銘柄を毎晩フル分析する。

## B. 強化1：出来高フィルター

シグナルが「出来高を伴っているか」で信頼度を変える。

### 計算

- 出来高移動平均 `vol_sma = SMA(volume, 20)`
- 当日出来高が平均の何倍か `vol_ratio = volume[today] / vol_sma`

### スコアリングへの反映（推奨ロジック）

- buy/sell シグナルが出ているとき、`vol_ratio >= 1.5`（出来高急増）なら **スコアに +1 のボーナス**
- `vol_ratio < 0.7`（閑散）のときは、そのシグナルの重みを**減衰**（例：スコア寄与を半分に）
- 倍率の閾値（1.5 / 0.7）は signal_config で調整可能にする

### signal_config への追加 rule_type

| rule_type | params 例 | 説明 |
|---|---|---|
| `volume_filter` | `{"sma":20,"surge":1.5,"quiet":0.7,"bonus":1}` | 出来高急増でボーナス、閑散で減衰 |

## C. 強化2：週足トレンド足切り（マルチタイムフレーム）

「上位足で方向を決め、日足で仕掛ける」。週足が下降中の銘柄は買い候補から外す。

### データ取得

- yfinance で **週足**も取得（`interval="1wk"`）。日足とは別に保持。
- 週足は戦略の方向決定に使う。日足がエントリー主判断、という役割分担。

### 判定ロジック

- 週足の `SMA(13)`（約3ヶ月）の傾きで大局トレンドを判定
  - 上向き → 買い許可
  - 下向き → **buyシグナルを抑制**（スコアが閾値を超えても direction を neutral に丸める、または -2 ペナルティ）
- sell側は逆（週足が上向きなら過度な売り判定を抑制）

### signal_config への追加 rule_type

| rule_type | params 例 | 説明 |
|---|---|---|
| `weekly_trend_filter` | `{"sma":13,"mode":"block"}` | mode=block: 逆行する向きのシグナルを無効化 / mode=penalty: 減点のみ |

> 注: 分足（60分/15分）は yfinance の取得期間制約（直近約60日）があるため、本追補では**柱にしない**。
> 将来「直近の指値精度向上」の補助としてのみ検討する。戦略の柱は日足＋週足。

## D. 強化3：ATRによる出口自動設計

「その銘柄が普段どれくらい動くか」を基準に、損切り・利確・指値を決める。固定%より理にかなう。

### 計算

- `ATR = Average True Range（期間14）`（pandas-taの `df.ta.atr(length=14)`）
- ATRは「1日の平均的な値動き幅」を表す

### 出口の算出（推奨デフォルト）

buyシグナルの銘柄について、判定日の終値 `close` を基準に：

| 項目 | 計算式（デフォルト） | 意味 |
|---|---|---|
| 損切りライン | `close - 1.5 * ATR` | 想定外に逆行したら撤退 |
| 利確目安 | `close + 2.0 * ATR` | リスクリワード ≒ 1 : 1.3 |
| （倍率は調整可） | `stop_mult` / `target_mult` | signal_config で変更 |

sellシグナル／保有銘柄の決済時も同様に対称で算出。

### signal_config への追加 rule_type

| rule_type | params 例 | 説明 |
|---|---|---|
| `atr_exit` | `{"length":14,"stop_mult":1.5,"target_mult":2.0}` | ATRベースの損切/利確算出 |

## E. 強化4：翌日の指値提案（作戦ボード）

判定（buy/sell）に加えて「**いくらで指すか**」まで提示する。夜のルーティンを「見る→その数字で注文を置く」だけにする。

### 提案指値の算出方法

複数の候補からアプリが提示する。デフォルトは**サポート/MA基準**、補助でATR基準。

| 指値の決め方 | 計算 | 用途 |
|---|---|---|
| サポート基準（買） | 過去N日(例20)の安値の少し上 | 押し目を待ち構える |
| MA基準（買） | 5日線 or 25日線の近辺 | 移動平均への引きつけ |
| ATR基準（買） | `close - 0.5 * ATR` | 前日終値からボラ分だけ下 |
| （売／決済） | 上記の対称、または成行/前日終値付近 | 保有の手仕舞い |

> デフォルトはサポート基準を採用し、MA基準・ATR基準は参考値として併記。
> どれを主にするかは設定で選べるようにする。

### 出力イメージ（作戦ボード）

```
【明日の作戦】2026-06-16 寄付き前
8306.T 三菱UFJ   判定: BUY (スコア+4 / 出来高1.8倍 / 週足↑)
  └ 提案指値(買): 1,820円  [サポート1,815 + 5日線1,822 の少し下]
  └ 利確目安: 1,890円 (+2.0ATR) / 損切: 1,780円 (-1.5ATR)
7203.T トヨタ     判定: NEUTRAL  → 様子見
9984.T SBG       判定: SELL (スコア-3 / 保有中)
  └ 提案指値(売): 成行 または 前日終値付近
```

## F. データモデルの差分（基本仕様書 §2 への追加）

### price_data の拡張

週足を別途持つため、足種を区別する。

```sql
-- 既存 price_data に timeframe 列を追加（または週足用テーブルを別途用意）
ALTER TABLE price_data ADD COLUMN timeframe TEXT DEFAULT 'D';  -- 'D'=日足, 'W'=週足
-- 主キーは (ticker, date, timeframe) 相当に
```

### 作戦ボード保存用テーブル（新規）

```sql
CREATE TABLE daily_plan (
  id            INTEGER PRIMARY KEY AUTOINCREMENT,
  ticker        TEXT NOT NULL,
  plan_date     TEXT NOT NULL,        -- 翌営業日
  direction     TEXT,                 -- 'buy'|'sell'|'neutral'
  score         INTEGER,
  vol_ratio     REAL,                 -- 出来高倍率
  weekly_trend  TEXT,                 -- 'up'|'down'|'flat'
  limit_price   REAL,                 -- 提案指値
  stop_price    REAL,                 -- 損切ライン
  target_price  REAL,                 -- 利確目安
  rationale     TEXT,                 -- 根拠（JSON or テキスト）
  created_at    TEXT DEFAULT (datetime('now'))
);
```

## G. シグナル判定フローの差分（基本仕様書 §3 への上書き）

`evaluate()` の処理順を以下に拡張する。

```
1. 日足で各指標を計算しベーススコアを算出（基本仕様書のまま）
2. 出来高フィルター適用（強化1）
     - vol_ratio に応じてボーナス/減衰
3. 週足トレンド足切り適用（強化2）
     - 週足の向きと逆行する direction を block / penalty
4. 最終 direction を確定（buy/sell/neutral）
5. direction が buy/sell の銘柄について（強化3・4）
     - ATR から stop_price / target_price を算出
     - サポート/MA/ATR から limit_price を算出
6. daily_plan に保存し、作戦ボードとして出力
```

> 重要: look-ahead bias を避ける。各営業日の判定にはその日までのデータのみ使う（基本仕様書 §7 の原則を週足・出来高・ATRにも適用）。

## J. シミュレーションの差分（基本仕様書 §4 への追加）

ペーパートレードに出口ルールを組み込む。

- エントリーは buy シグナル＋提案指値で約定したとみなす（指値に当日安値が到達したら約定、など簡易判定でよい）
- 保有中は **stop_price / target_price** に当日の高値・安値が到達したら決済
- これにより「シグナルの良し悪し」だけでなく「出口設計の良し悪し」も検証できる

成績項目（基本仕様書 §4）に加えて：

- 利確で終わった回数 / 損切で終わった回数
- 平均保有日数
- リスクリワード実績（平均利益 ÷ 平均損失）

## K. ロードマップへの差し込み（基本仕様書 §6 との対応）

基本仕様書の Phase に対し、本追補は以下に差し込む。

| 追補項目 | 差し込み先 |
|---|---|
| 出来高フィルター（B） | Phase 0〜1（早期に。OHLCVだけで完結し効果大） |
| 週足トレンド足切り（C） | Phase 1（週足取得の追加） |
| ATR出口設計（D） | Phase 1 |
| 作戦ボード・指値提案（E〜I） | Phase 2.5（通知の後、UIの前後）として新設 |
| 出口入りシミュレーション（J） | Phase 4（運用検証時に活用） |

### 着手順の推奨

基本仕様書どおり **Phase 0（日足スコアリングの検証）** をまず通す。
そこに **出来高フィルター** を最初から混ぜておくと、検証段階から精度が上がる。
週足・ATR・作戦ボードは Phase 1 以降で順に足す。

## L. 変更しない原則（再確認）

- 自動売買はしない（提案指値も「人間が置くための数字」。発注処理は実装しない）。
- データ源は yfinance のみ。決算日程・ニュース・板情報は対象外。
- 日足＋週足が柱。分足はyfinanceの期間制約により柱にしない。
- look-ahead bias を避ける（週足・出来高・ATRにも適用）。
- 投資は自己責任・シグナルは予測を保証しない旨の注意表示を入れる。
